При кодировании цвета в устаревшем теге FONT можно указывать некоторые забавные значения, например:
<font color="#BruceWillis">... </font>, который дает светлозеленый цвет (Hello)
<font color="#CANDY">...<font>, который дает красный цвет (Hello)

Так получается из-за того, что действует следующий алгоритм:

для #BruceWillis
  • Заменяем все неправильные шестнадцатеричные символы нулями 0, BruceWillis превращается в b00ce0i00i0
  • Добиваем” число нулями до количества знаков, делимого на 3 (11 -> 12), получаем b00c e0i0 0i00
  • Разбиваем на три группы, каждая из которых отвечает за одну компоненту RGB: RGB (b00c, e0i0, 0i00)
  • Обрезаем каждый из аргументов справа, оставляя 2 символа, и получаем наш результат RGB (b0, e0, 0i) = #b0e00i или он же RGB(0, 224, 157).
для #CANDY
  • Заменяем все неправильные шестнадцатеричные символы нулями 0, CANDY превращается в ca0d0
  • Добиваем” число нулями до количества знаков, делимого на 3 (5 -> 6), получаем ca0d00
  • Разбиваем на три группы, каждая из которых отвечает за одну компоненту RGB: RGB (ca,0d,00)
  • Пполучаем наш результат RGB (b0, e0, 0i) = #ca0d00 или он же RGB(202, 13, 0).
